CSCI 272 — Object-Oriented Programming (C++)

Course portal for students: weekly plan, assessments, expectations, tools, and quick links.

Overview

This course builds practical C++ skills through object-oriented design: classes, composition, operator overloading, inheritance, and polymorphism — plus robust I/O with streams, stringstreams, and files. You’ll write clean, modular code and be able to explain your decisions.

Instructor: Avijit Roy • John Jay College of Criminal Justice (CUNY)

Syllabus (PDF) Weekly Plan Assessments Tools

Learning Outcomes

  • Design and implement classes with clean interfaces.
  • Use vectors, strings, templates, and exceptions appropriately.
  • Work confidently with streams, stringstreams, and file I/O.
  • Apply inheritance and polymorphism to real problems.
  • Explain and defend design decisions and code behavior.

Weekly Plan (Snapshot)

A condensed roadmap. Exact pacing may adjust slightly — check in class for updates.

Weeks 1–3 • Foundations

  • Review: functions, arrays, pointers → add vectors & templates.
  • Strings: build, compare, concatenate, convert; practical parsing tasks.

Weeks 4–6 • Classes & Streams

  • Intro to classes: public/private, member functions, header/implementation split.
  • Streams & recovery from bad input; stringstreams for parsing/building text.

Week 7 • Midterm

  • Written exam + individual OOP project (Weeks 1–6 scope).

Weeks 8–10 • Files & Operator Overloading

  • Text/CSV file I/O; modular design with headers/implementations.
  • Overload operators (unary/binary) with correct semantics.

Weeks 11–12 • Inheritance & Polymorphism

  • Base/derived classes; protected; virtual functions & abstract classes.

Weeks 13–14 • Templates & Exceptions

  • Class templates for reusable designs; try/catch/throw and custom exceptions.

Week 15 • Final

  • Team capstone: plan, implement, present a complete OOP C++ application.

Assessments

Participation & Quizzes

Short checks to reinforce weekly topics; start on time. No make-ups for late arrival unless documented emergencies.

  • In-class discussion & code walk-throughs
  • Two announced quizzes + brief pop quizzes

Assignments

Frequent, focused practice. Expect clean headers, comments, and testable examples. Submit on time.

Midterm

  • Written exam on concepts & code analysis.
  • Individual take-home OOP project (Weeks 1–6).

Final — Team Capstone

  • Plan (UML encouraged), implement, and present a complete OOP app.
  • Clear division of work; everyone explains their code.

Expectations

Quick FAQs

How do I submit assignments?

Follow instructions on Brightspace for each assignment. Use separate headers/implementations and include brief comments explaining your logic.

What happens if my code compiles but fails some tests?

We focus on correctness, clarity, and adherence to requirements. Partial credit may be limited if logic is incorrect or unexplained.

Can I use AI tools?

Use AI to learn concepts, not to replace your work. Cite any use. You must be able to explain your code in class.
